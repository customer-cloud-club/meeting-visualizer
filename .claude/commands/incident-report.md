# インシデントレポート作成

インシデントレポートを作成します。

## パラメータ

- `$1`: インシデントID（必須）

## 実行

incident agentを起動して、以下の情報を収集・整理してください：

1. 発生日時と検知経緯
2. 影響範囲とユーザー影響
3. 対応タイムライン
4. 根本原因分析（RCA）
5. 再発防止策

## テンプレート

```markdown
# インシデントレポート: ${1}

## サマリー
- 発生: YYYY-MM-DD HH:MM
- 復旧: YYYY-MM-DD HH:MM
- 影響: [影響を受けたユーザー数/サービス]

## 詳細
[詳細な経緯]

## 再発防止策
- [ ] アクション1
- [ ] アクション2
```
