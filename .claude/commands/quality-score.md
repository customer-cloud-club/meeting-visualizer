# /quality-score - 品質スコア算出

コードの品質を総合的に評価し、スコアを算出します。

## 使い方

```bash
# プロジェクト全体の品質スコア
/quality-score

# 特定ディレクトリの品質スコア
/quality-score --path=src/auth

# 詳細レポート出力
/quality-score --verbose --output=report.json
```

## 評価項目

1. **TypeScript型安全性** (25点)
   - TypeScriptエラー数
   - `any`型の使用率
   - 型カバレッジ

2. **テストカバレッジ** (25点)
   - ライン・ブランチ・関数カバレッジ
   - テストケース数

3. **ESLint準拠** (20点)
   - ESLintエラー/警告数
   - コードスタイル統一性

4. **セキュリティ** (20点)
   - 脆弱性スキャン
   - セキュリティベストプラクティス

5. **ドキュメント** (10点)
   - コメント率
   - READMEの充実度

## スコア基準

| スコア | 判定 | 説明 |
|--------|------|------|
| 90-100 | 優秀 | 非常に高品質 |
| 80-89 | 合格 | 高品質（CCAGIの基準） |
| 70-79 | 要改善 | 改善が必要 |
| 60-69 | 警告 | 重大な問題あり |
| 0-59 | 不合格 | デプロイ不可 |
